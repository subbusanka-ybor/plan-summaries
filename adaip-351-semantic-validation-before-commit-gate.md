# ADAIP-351: Semantic Validation Before Commit Gate

## Context

Cortex generates deployment manifests (K8s, CI, Dockerfile) from static templates with variable substitution. The existing `CommitValidator` enforces **structural** correctness — directory layout, required fields, IMAGE_NAME suffix rules. But it cannot catch **semantic** mismatches between generated files and the actual source code.

This gap was discovered reviewing cortex-mcp PR #2 (generated by Cortex for the cortex-mcp service). The templates produced correct output for standard apps (e.g., sale-service), but for non-standard apps the defaults went unchallenged:
- Health probe defaulted to `/health/ready` — but the app exposes `/health`
- Port defaulted to `8080` — correct for this case, but unverified
- Config block was empty — but the app uses several env vars (`DATABASE_URL`, `ANTHROPIC_API_KEY`, etc.)

**Goal**: Add a semantic validation pass that reads the source repo and blocks the commit when drift is found — with file:line references and remediation steps so Claude can self-correct.

## Approach

`SemanticValidator` class in `commit_validator.py` runs as a **pre-commit gate** (not advisory). If drift is found, the commit is blocked and Claude receives a drift report with exact source locations and fix instructions.

### Integration point

In `CommitGitHubFiles.execute()` (`github_write.py`), after structural validation passes and **before** `client.commit_files()`:

```python
semantic = SemanticValidator()
semantic_result = await semantic.validate(
    files=files, owner=owner, repo=repo, context=context,
)
if semantic_result.violations:
    return ToolResult(content=format_semantic_warnings(semantic_result.violations), is_error=True)
```

## Checks

### Check 1: Health probe path verification
- Parse generated `application.yaml` for `readinessProbe` → `path`
- Fetch the repo tree via `GitHubClient.get_tree_recursive()`
- Scan Python files for route decorators with **line numbers** (`@app.get("/health/ready")`, `@router.get(...)`, Flask `@app.route(...)`)
- **Block commit** if the declared health path doesn't appear in any route decorator

### Check 2: Port consistency
- Parse generated `application.yaml` for `deployment.ports[0].port`
- Parse the Dockerfile for `EXPOSE` directives and CMD `--port` args with **line numbers**
- **Block commit** on mismatch

### Check 3: Config completeness (env var coverage)
- Parse `spec.config` section for declared env var names
- Use AST-based `_parse_env_refs_with_lines()` to find env vars with **line numbers** in source code
- **Block commit** about env vars used in source but not declared in config or secrets
- Filter out standard runtime vars (`PATH`, `HOME`, `PYTHONPATH`, `NODE_ENV`, etc.)

## Drift Report Format

Each finding produces a structured drift report:

```
COMMIT BLOCKED — semantic drift detected between manifests and source code.

FINDINGS:
  1. [health-probe] Health probe path drift detected.
       Template generated: readinessProbe.httpGet.path = '/health/ready'
       Source code has:    routes defined at:
         /health  (found at main.py:3)
         /api/items  (found at main.py:5)
       File to fix: .platform/kubernetes/base/application.yaml → spec.deployment.readinessProbe.httpGet.path
       Fix: call extract_deployment_variables with health_endpoint matching an actual route, then re-render.

  2. [port] Container port drift detected.
       Template generated: deployment.ports[0].port = 8080
       Source code has:    EXPOSE 3000 (found at Dockerfile:2)
       File to fix: .platform/kubernetes/base/application.yaml → spec.deployment.ports[0].port
       Fix: call extract_deployment_variables with port=3000, then re-render.

  3. [config] Config/secrets drift detected.
       Template generated: spec.config has 1 var(s)
       Source code has:    2 additional env var(s) not in manifest:
         ANTHROPIC_API_KEY  (found at config.py:3)
         DATABASE_URL  (found at config.py:2)
       File to fix: .platform/kubernetes/base/application.yaml → spec.config / spec.secrets
       Fix: run detect_secrets_usage to classify these, then re-render.

REMEDIATION: Fix the parameters in extract_deployment_variables (health_endpoint, port, config vars),
then call render_deployment_manifests again, then retry commit_github_files.
```

## Key Design Decisions

- **Pre-commit gate, not advisory** — drift blocks the commit so Claude must fix before proceeding
- **file:line references** — every finding includes exact source locations for route decorators, EXPOSE directives, and env var references
- **Structured drift report** — "Template generated X, Source code has Y (found at file:line), File to fix: manifest → YAML.path"
- **Single `get_tree_recursive()` call** shared across all checks
- **Line-aware parsers**: `_extract_routes_with_lines()` (regex per-line), `_parse_env_refs_with_lines()` (AST with `node.lineno`)
- **Graceful degradation** — if source repo is inaccessible, skip semantic checks silently
